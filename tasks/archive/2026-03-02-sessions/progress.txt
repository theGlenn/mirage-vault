## Codebase Patterns
- Tauri 2 app: Rust backend in `mirage-vault/src-tauri/src/`, frontend in `mirage-vault/src/`
- Frontend TypeScript modules live in `mirage-vault/src/lib/` (SvelteKit `$lib` alias)
- Run tests with `npx tsx --test <file>` from the `mirage-vault/` directory
- Detection types defined in `src/lib/detectors/types.ts`: `Detection` interface and `EntityType` union
- Must `use tauri::Manager;` to access `.manage()` and `.path()` on App/AppHandle
- Database state uses `Mutex<Connection>` wrapped in a struct: `DbState(pub Mutex<Connection>)`
- SQLite DB stored in Tauri app data dir (`app.path().app_data_dir()`)
- Use `CREATE TABLE IF NOT EXISTS` for idempotent schema creation
- Idempotent migrations: `ALTER TABLE ADD COLUMN` wrapped in `match` with `duplicate column name` error catch
- `cargo check` runs from `mirage-vault/src-tauri/`
- Foreign keys enabled via `PRAGMA foreign_keys = ON`
- Tauri commands use `State<'_, DbState>` for DB access; lock mutex with `db.0.lock()`
- Commands return `Result<T, String>` â€” map rusqlite errors with `.map_err(|e| e.to_string())`
- Serde structs: `Deserialize` for inputs, `Serialize` for outputs
- Commands registered in `lib.rs` via `tauri::generate_handler![]` with `commands::` prefix
- Tauri `invoke()` uses camelCase parameter names (e.g., `sessionId`, `rawContent`), which map to snake_case Rust params automatically
- `refreshItems()` pattern: call `invoke<T[]>('list_items')` after any mutation to keep UI in sync
- Theme store: import `theme` store and `initTheme()` from `$lib/stores/theme`
- Design tokens: use CSS custom properties like `var(--bg-primary)` everywhere
- PixelIcon component: `<PixelIcon name="folder" size={24} />` â€” import `IconName` type for type-safe icon names
- Sidebar component: `<Sidebar activeView={activeView} onnavigate={(view) => ...} />` â€” exports `ActiveView` type
- Svelte 5 event pattern: use `onnavigate` callback prop instead of `createEventDispatcher`
- All styles use CSS custom properties from tokens.css â€” no hardcoded colors
- Border-radius 0px throughout for pixel art sharpness; borders use `2px solid` for pixel feel
- FileCard exports `VaultItem` interface â€” import from `$lib/components/FileCard.svelte`
- FileBrowser component: callback props for events, renders empty state or file grid
- SearchBar: `<SearchBar bind:value={searchQuery} />` â€” uses `$bindable()` for two-way binding
- Center `<main>` has `position: relative` â€” required for absolute positioning overlays
- Center-area overlays: `position: absolute; inset: 0` constrained to center panel (not viewport)
- FileViewer exports `ItemDetail` interface â€” import from `$lib/components/FileViewer.svelte`
- EntityPanel exports `groupEntitiesByType()` â€” reusable entity grouping logic
- Svelte 5 `$derived.by(() => ...)` for complex derived values
- `<script lang="ts" module>` for exported types/functions importable from `.svelte` files
- Escape key dismissal: use `<svelte:window onkeydown={handler} />` for overlay components
- ConfirmDialog: callback props `onconfirm` and `oncancel`, constrained to center area
- `color-mix(in srgb, var(--bg-surface) 80%, transparent)` for semi-transparent backgrounds
- Crypto: `crypto::encrypt(key, plaintext)` and `crypto::decrypt(key, ciphertext)` â€” key from `CryptoState` managed state
- Encryption fallback: if decryption fails, return raw string (handles pre-encryption data)
- SessionList component: exports `SessionSummary` interface, uses `onselectsession` callback prop
- ConfirmDialog can be rendered inside child components (not just +page.svelte) since overlay uses `position: absolute; inset: 0`
- SessionDetail component: `<SessionDetail sessionId={id} onback={() => ...} />` â€” loads session data on mount via `invoke('get_session')`
- SessionEntryCard component: `<SessionEntryCard entry={e} itemName={name} onselect={(id) => ...} />` â€” exports `SessionEntry` interface
- Output entries use `--color-grim-blue` accent, input entries use `--accent-orange` â€” consistent complementary color scheme
- Inline editing pattern: click text â†’ show input, blur/enter saves, escape cancels â€” reuse for editable labels
- Frosted glass fixed-bottom: `position: absolute; bottom: 0; background: color-mix(in srgb, var(--bg-surface) 80%, transparent); backdrop-filter: blur(8px);` â€” parent needs `position: relative`, sibling scroll area needs bottom padding
- VaultPicker component: `<VaultPicker sessionItemIds={set} onselect={(id) => ...} onclose={() => ...} />` â€” self-contained, calls `invoke('list_items')` internally
- Session item linking flow: `add_item_to_session` â†’ `reconcile_item_tokens` â†’ `get_item` â†’ `add_session_entry` (input type)
- Tauri drag-drop context routing: check `activeView` and `selectedSessionId` in the global drop handler to route to session-aware or browse-aware ingestion
- `export function` in Svelte 5 instance script exposes methods on component instance via `bind:this`
- SessionEntryPopup component: `<SessionEntryPopup entry={e} sessionId={id} sessionEntities={entities} onclose={() => ...} onrefresh={() => ...} />` â€” overlay with masked/decoded toggle and token highlighting
- Session entities for token mapping: `invoke('get_session_entities', { sessionId })` returns `SessionEntityOutput[]` with `entity_type`, `original_value`, `token`

## 2026-03-01 - US-005
- Implemented two Tauri commands: `add_session_entry`, `decode_session_entry`
- `add_session_entry` encrypts raw_content, inserts into session_entries, updates sessions.updated_at, returns new entry id
- `decode_session_entry` loads all entities across session items via session_items join, builds tokenâ†’original_value map, decrypts entry's raw_content, uses regex `\[\[(\w+)[_:](\w+)\]\]` to match both `[[TYPE:HASH]]` and `[[TYPE_N]]` formats, replaces tokens via `replace_all` with closure lookup, encrypts decoded result, stores in decoded_content column, returns decoded text
- Tokens with no match in the session vault are left as-is (partial decode is valid)
- Added `use regex::Regex` import (regex crate was already in Cargo.toml)
- Registered both commands in lib.rs generate_handler![]
- Files changed: `mirage-vault/src-tauri/src/commands.rs`, `mirage-vault/src-tauri/src/lib.rs`
- **Learnings for future iterations:**
  - `regex::Regex::new()` returns a Result â€” map_err for consistent error handling
  - `replace_all` with a closure is the idiomatic way to do conditional replacements â€” the closure receives `Captures` and returns the replacement string
  - Token map key should be the full token string (e.g., `[[EMAIL:a3f2c1b9]]`) since that's what appears in the text
  - Use `HashMap::entry().or_insert()` to keep the first token mapping when duplicates exist

## 2026-03-01 - US-006
- Implemented `reconcile_item_tokens(session_id, item_id)` Tauri command for cross-file token namespace reconciliation
- Algorithm: loads other session items' entities â†’ builds (type, value) â†’ token map â†’ loads new item's entities â†’ reuses existing tokens for matching (type, value) pairs â†’ generates new non-conflicting [[TYPE:HASH]] tokens for unique entities â†’ rewrites masked_content with token replacements â†’ updates entities and hash_mappings atomically in a transaction
- Token replacement sorts by old_token length descending to prevent partial match issues
- New tokens use `rand::thread_rng()` to generate 8-char hex hashes in [[TYPE:HASH]] format
- Idempotent: running twice produces same result since tokens already match on second pass
- Added `use rand::Rng` and changed `HashMap` import to `std::collections::{HashMap, HashSet}`
- Registered command in lib.rs generate_handler![]
- Files changed: `mirage-vault/src-tauri/src/commands.rs`, `mirage-vault/src-tauri/src/lib.rs`
- **Learnings for future iterations:**
  - For token reconciliation, exclude the current item from the session entity query (`si.item_id != ?2`) since the item is already linked via `add_item_to_session`
  - Register new tokens in `session_map` during iteration so duplicate (type, value) entities within the same item get the same token
  - Sort token replacements by length descending before string replacement to prevent partial matches (e.g., `[[EMAIL_11]]` before `[[EMAIL_1]]`)
  - `rand::thread_rng().gen::<[u8; 4]>()` + manual hex formatting avoids needing the `hex` crate
  - The `extract_hash_from_token` helper already exists in commands.rs for parsing [[TYPE:HASH]] tokens
