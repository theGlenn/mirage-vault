## Codebase Patterns
- Tauri 2 app: Rust backend in `mirage-vault/src-tauri/src/`, frontend in `mirage-vault/src/`
- Frontend TypeScript modules live in `mirage-vault/src/lib/` (SvelteKit `$lib` alias)
- `@types/node` is installed for `node:test` and `node:assert` type support
- Run tests with `npx tsx --test <file>` from the `mirage-vault/` directory
- Detection types defined in `src/lib/detectors/types.ts`: `Detection` interface and `EntityType` union
- Must `use tauri::Manager;` to access `.manage()` and `.path()` on App/AppHandle
- Database state uses `Mutex<Connection>` wrapped in a struct: `DbState(pub Mutex<Connection>)`
- SQLite DB stored in Tauri app data dir (`app.path().app_data_dir()`)
- Use `CREATE TABLE IF NOT EXISTS` for idempotent schema creation
- `detect()` from `$lib/detectors` is the unified entry point for all entity detection â€” import from `$lib/detectors` (not regex/nlp directly)
- Toolbar action buttons use `.toolbar-btn` class; add new buttons alongside Copy in the `.viewer-toolbar` (right side via `justify-content: space-between`)
- Toast notifications: set `toastMessage` state string + `setTimeout` to clear; reuse for any brief user feedback
- `cargo check` runs from `mirage-vault/src-tauri/`
- Inside `<pre>` tags, Svelte template blocks (`{#each}`, `{#if}`) must be on one line to avoid injecting whitespace
- compromise `.people()` and `.organizations()` return entity text; use `indexOf()` loop to get positions in source text
- NLP tests should use `>=` assertions since compromise may detect additional entities beyond expectations
- Foreign keys enabled via `PRAGMA foreign_keys = ON`
- Tauri commands use `State<'_, DbState>` for DB access; lock mutex with `db.0.lock()`
- Commands return `Result<T, String>` â€” map rusqlite errors with `.map_err(|e| e.to_string())`
- Serde structs: `Deserialize` for inputs, `Serialize` for outputs
- Commands registered in `lib.rs` via `tauri::generate_handler![]` with `commands::` prefix
- Tauri plugins: register in `lib.rs` via `.plugin()`, add permissions in `capabilities/default.json`
- Native dialogs: use `tauri_plugin_dialog` â€” import `DialogExt` trait, call `app.dialog().file()` for file dialogs
- Async Tauri commands with `pub async fn` when using dialog blocking methods
- `blocking_save_file()` returns `Option<FilePath>` â€” use `.as_path().unwrap()` to get `&Path`
- Tauri `invoke()` uses camelCase parameter names (e.g., `fileType`, `rawContent`), which map to snake_case Rust params automatically
- `refreshItems()` pattern: call `invoke<T[]>('list_items')` after any mutation to keep sidebar in sync
