## Codebase Patterns
- Tauri 2 app: Rust backend in `aether-vault/src-tauri/src/`, frontend in `aether-vault/src/`
- Frontend TypeScript modules live in `aether-vault/src/lib/` (SvelteKit `$lib` alias)
- `@types/node` is installed for `node:test` and `node:assert` type support
- Run tests with `npx tsx --test <file>` from the `aether-vault/` directory
- Detection types defined in `src/lib/detectors/types.ts`: `Detection` interface and `EntityType` union
- Must `use tauri::Manager;` to access `.manage()` and `.path()` on App/AppHandle
- Database state uses `Mutex<Connection>` wrapped in a struct: `DbState(pub Mutex<Connection>)`
- SQLite DB stored in Tauri app data dir (`app.path().app_data_dir()`)
- Use `CREATE TABLE IF NOT EXISTS` for idempotent schema creation
- `detect()` from `$lib/detectors` is the unified entry point for all entity detection â€” import from `$lib/detectors` (not regex/nlp directly)
- Toolbar action buttons use `.toolbar-btn` class; add new buttons alongside Copy in the `.viewer-toolbar` (right side via `justify-content: space-between`)
- Toast notifications: set `toastMessage` state string + `setTimeout` to clear; reuse for any brief user feedback
- `cargo check` runs from `aether-vault/src-tauri/`
- Inside `<pre>` tags, Svelte template blocks (`{#each}`, `{#if}`) must be on one line to avoid injecting whitespace
- compromise `.people()` and `.organizations()` return entity text; use `indexOf()` loop to get positions in source text
- NLP tests should use `>=` assertions since compromise may detect additional entities beyond expectations
- Foreign keys enabled via `PRAGMA foreign_keys = ON`
- Tauri commands use `State<'_, DbState>` for DB access; lock mutex with `db.0.lock()`
- Commands return `Result<T, String>` â€” map rusqlite errors with `.map_err(|e| e.to_string())`
- Serde structs: `Deserialize` for inputs, `Serialize` for outputs
- Commands registered in `lib.rs` via `tauri::generate_handler![]` with `commands::` prefix
- Tauri plugins: register in `lib.rs` via `.plugin()`, add permissions in `capabilities/default.json`
- Native dialogs: use `tauri_plugin_dialog` â€” import `DialogExt` trait, call `app.dialog().file()` for file dialogs
- Async Tauri commands with `pub async fn` when using dialog blocking methods
- `blocking_save_file()` returns `Option<FilePath>` â€” use `.as_path().unwrap()` to get `&Path`
- Tauri `invoke()` uses camelCase parameter names (e.g., `fileType`, `rawContent`), which map to snake_case Rust params automatically
- lopdf: use `Document::load_from(Cursor::new(bytes))` for in-memory PDF loading; `doc.get_pages()` returns sorted `BTreeMap<u32, ObjectId>`; `doc.extract_text(&[page])` returns text per page
- lopdf `replace_text()` takes 3 args: `(page_number: u32, text: &str, other_text: &str)` â€” must iterate per page; `save_to()` takes `&mut W: Write`, use `File::create()` then pass `&mut file`
- `refreshItems()` pattern: call `invoke<T[]>('list_items')` after any mutation to keep sidebar in sync
- Page marker regex: `/\n*--- Page \d+ ---\n*/g` â€” use `\n*` to capture surrounding newlines; strip them from display text
- `splitByPageMarkers()` and `getEntitiesForBlock()` in `+page.svelte` handle per-block entity highlighting for PDF page markers
- Content viewer uses `<div class="viewer-content">` with `white-space: pre-wrap` â€” equivalent to `<pre>` but allows block-level child elements for page markers
- Rust `Vec<u8>` serde-serializes to a JSON number array; frontend type is `number[]`, and can be sent back to Tauri commands directly without base64
- Theme store: import `theme` store and `initTheme()` from `$lib/stores/theme`; call `initTheme()` in root `+layout.svelte` `onMount`
- Design tokens: import `$lib/styles/tokens.css` in root `+layout.svelte`; use CSS custom properties like `var(--bg-primary)` everywhere
- `+layout.svelte` uses Svelte 5 `$props()` with `children` snippet and `{@render children()}`
- Inline theme script in `app.html` prevents flash of unstyled content â€” reads `localStorage('aether-theme')` before Svelte hydrates
- PixelIcon component: `<PixelIcon name="folder" size={24} />` â€” import `IconName` type from `$lib/components/PixelIcon.svelte` for type-safe icon names
- Components directory: `src/lib/components/` â€” all shared UI components live here
- Sidebar component: `<Sidebar activeView={activeView} onnavigate={(view) => ...} />` â€” exports `ActiveView` type
- Svelte 5 event pattern: use `onnavigate` callback prop instead of `createEventDispatcher`; parent passes function directly
- All styles now use CSS custom properties from tokens.css â€” no hardcoded colors in +page.svelte
- Border-radius 0px throughout for pixel art sharpness; borders use `2px solid` for pixel feel
- Cannot nest `<button>` inside `<button>` â€” Svelte 5 enforces valid HTML nesting; use `<div>` with onclick + `<!-- svelte-ignore a11y_click_events_have_key_events a11y_no_static_element_interactions -->` for clickable card wrappers
- FileCard exports `VaultItem` interface â€” import from `$lib/components/FileCard.svelte`; file type icon mapping: txtâ†’file, pdfâ†’file-alt, csvâ†’table, jsonâ†’code, mdâ†’article
- Right sidebar conditionally rendered with `{#if activeView === 'browse'}` â€” hidden during settings view
- Settings view clears selected file: `selectedItemId = null; selectedItem = null` on navigate to settings
- FileBrowser component: `<FileBrowser {items} {processing} {errorMessage} {dragOver} bind:pasteText ... />` â€” accepts callback props for events, renders empty state or file grid
- `VaultItem` interface is exported from `FileCard.svelte` â€” import it from `$lib/components/FileCard.svelte` in all components that need it
- `$bindable()` in Svelte 5: use for two-way binding on component props (e.g., `pasteText = $bindable('')`)
- SearchBar: `<SearchBar bind:value={searchQuery} />` â€” uses `$bindable()` for two-way binding, positioned absolute at bottom of center area
- Center `<main>` has `position: relative` â€” required for absolute positioning of SearchBar and future FileViewer overlay
- `$derived()` for computed filtering: `let filteredItems = $derived(...)` â€” Svelte 5 reactive computed values
- `color-mix(in srgb, var(--bg-surface) 80%, transparent)` for semi-transparent backgrounds with CSS custom properties (no hardcoded rgba)
- Center-area overlays: `position: absolute; inset: 0` on the overlay + parent `<main>` has `position: relative` â€” used by FileViewer and ConfirmDialog
- FileViewer exports `ItemDetail` interface â€” import from `$lib/components/FileViewer.svelte` for item detail types
- EntityPanel component: `<EntityPanel {activeView} {selectedItem} {items} />` â€” self-contained right sidebar with context-dependent content
- EntityPanel exports `groupEntitiesByType()`, `EntityGroup`, `EntityGroupItem` â€” reusable entity grouping logic moved from +page.svelte
- Svelte 5 `$derived.by(() => ...)` for complex derived values that need multi-line computation (vs `$derived(expr)` for simple expressions)
- `<script lang="ts" module>` for exported types/functions that need to be importable from `.svelte` files â€” separate from instance `<script lang="ts">`
- Escape key dismissal: use `<svelte:window onkeydown={handler} />` for global keyboard shortcuts in overlay components
- SettingsScreen component: `<SettingsScreen />` â€” self-contained, reads/writes the theme store directly; no props needed
- Segmented control pattern: flex container with 2px solid border, child buttons separated by `border-right`, active option gets `--accent-orange` background + white text
- ConfirmDialog component: `<ConfirmDialog message="..." confirmLabel="Delete" cancelLabel="Cancel" onconfirm={...} oncancel={...} />` â€” constrained to center area with `position: absolute; inset: 0`
- ConfirmDialog uses `{@html message}` to support inline HTML (e.g., `<strong>`) in the message prop
- FileBrowser toolbar: when items exist, a toolbar header renders with Select Files, Paste Text toggle, and Export All buttons; pass `onexportall` and `exportingAll` props from +page.svelte
- Expandable paste panel: `showPastePanel` local state in FileBrowser toggles a collapsible textarea+button area below the toolbar â€” avoids cluttering the grid while keeping paste accessible
- Drag-over on file grid: `class:file-browser-dragover={dragOver}` adds pulsing orange border to `.file-browser` div (reuses `pulse-border` keyframes from drop zone)
- Toast pixel art styling: `border: 2px solid var(--border-accent)`, `text-transform: uppercase`, `letter-spacing: 0.05em` for pixel art feel
