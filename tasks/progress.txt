## Codebase Patterns
- Tauri 2 app: Rust backend in `mirage-vault/src-tauri/src/`, frontend in `mirage-vault/src/`
- Frontend TypeScript modules live in `mirage-vault/src/lib/` (SvelteKit `$lib` alias)
- Run tests with `npx tsx --test <file>` from the `mirage-vault/` directory
- Detection types defined in `src/lib/detectors/types.ts`: `Detection` interface and `EntityType` union
- Must `use tauri::Manager;` to access `.manage()` and `.path()` on App/AppHandle
- Database state uses `Mutex<Connection>` wrapped in a struct: `DbState(pub Mutex<Connection>)`
- SQLite DB stored in Tauri app data dir (`app.path().app_data_dir()`)
- Use `CREATE TABLE IF NOT EXISTS` for idempotent schema creation
- Idempotent migrations: `ALTER TABLE ADD COLUMN` wrapped in `match` with `duplicate column name` error catch
- `cargo check` runs from `mirage-vault/src-tauri/`
- Foreign keys enabled via `PRAGMA foreign_keys = ON`
- Tauri commands use `State<'_, DbState>` for DB access; lock mutex with `db.0.lock()`
- Commands return `Result<T, String>` — map rusqlite errors with `.map_err(|e| e.to_string())`
- Serde structs: `Deserialize` for inputs, `Serialize` for outputs
- Commands registered in `lib.rs` via `tauri::generate_handler![]` with `commands::` prefix
- Tauri `invoke()` uses camelCase parameter names (e.g., `sessionId`, `rawContent`), which map to snake_case Rust params automatically
- `refreshItems()` pattern: call `invoke<T[]>('list_items')` after any mutation to keep UI in sync
- Theme store: import `theme` store and `initTheme()` from `$lib/stores/theme`
- Design tokens: use CSS custom properties like `var(--bg-primary)` everywhere
- PixelIcon component: `<PixelIcon name="folder" size={24} />` — import `IconName` type for type-safe icon names
- Sidebar component: `<Sidebar activeView={activeView} onnavigate={(view) => ...} />` — exports `ActiveView` type
- Svelte 5 event pattern: use `onnavigate` callback prop instead of `createEventDispatcher`
- All styles use CSS custom properties from tokens.css — no hardcoded colors
- Border-radius 0px throughout for pixel art sharpness; borders use `2px solid` for pixel feel
- FileCard exports `VaultItem` interface — import from `$lib/components/FileCard.svelte`
- FileBrowser component: callback props for events, renders empty state or file grid
- SearchBar: `<SearchBar bind:value={searchQuery} />` — uses `$bindable()` for two-way binding
- Center `<main>` has `position: relative` — required for absolute positioning overlays
- Center-area overlays: `position: absolute; inset: 0` constrained to center panel (not viewport)
- FileViewer exports `ItemDetail` interface — import from `$lib/components/FileViewer.svelte`
- EntityPanel exports `groupEntitiesByType()` — reusable entity grouping logic
- Svelte 5 `$derived.by(() => ...)` for complex derived values
- `<script lang="ts" module>` for exported types/functions importable from `.svelte` files
- Escape key dismissal: use `<svelte:window onkeydown={handler} />` for overlay components
- ConfirmDialog: callback props `onconfirm` and `oncancel`, constrained to center area
- `color-mix(in srgb, var(--bg-surface) 80%, transparent)` for semi-transparent backgrounds
- Crypto: `crypto::encrypt(key, plaintext)` and `crypto::decrypt(key, ciphertext)` — key from `CryptoState` managed state
- Encryption fallback: if decryption fails, return raw string (handles pre-encryption data)
