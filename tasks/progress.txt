## Codebase Patterns
- MCP sidecar state: `McpServerState { child: Mutex<Option<Child>>, passphrase: Mutex<Option<String>> }` — managed in lib.rs setup
- `stop_mcp_process()` is public for use in lib.rs `RunEvent::Exit` handler — keep it public
- `init_encryption` and `clear_encryption_key` accept `State<McpServerState>` — Tauri auto-injects, no frontend changes needed
- `find_mcp_script()` checks env var → dev path (CARGO_MANIFEST_DIR) → production path (next to exe) — add new paths as needed
- App exit cleanup: use `.build().run(|handle, event|)` pattern instead of `.run(context)` to handle `RunEvent::Exit`
- Tauri 2 app: Rust backend in `mirage-vault/src-tauri/src/`, frontend in `mirage-vault/src/`
- Frontend TypeScript modules live in `mirage-vault/src/lib/` (SvelteKit `$lib` alias)
- Run tests with `npx tsx --test <file>` from the `mirage-vault/mcp-server/` directory
- Detection types defined in `src/lib/detectors/types.ts`: `Detection` interface and `EntityType` union
- Must `use tauri::Manager;` to access `.manage()` and `.path()` on App/AppHandle
- Database state uses `Mutex<Connection>` wrapped in a struct: `DbState(pub Mutex<Connection>)`
- SQLite DB stored in Tauri app data dir (`app.path().app_data_dir()`)
- Use `CREATE TABLE IF NOT EXISTS` for idempotent schema creation
- Idempotent migrations: `ALTER TABLE ADD COLUMN` wrapped in `match` with `duplicate column name` error catch
- `cargo check` runs from `mirage-vault/src-tauri/`
- Foreign keys enabled via `PRAGMA foreign_keys = ON`
- Tauri commands use `State<'_, DbState>` for DB access; lock mutex with `db.0.lock()`
- Commands return `Result<T, String>` — map rusqlite errors with `.map_err(|e| e.to_string())`
- Serde structs: `Deserialize` for inputs, `Serialize` for outputs
- Commands registered in `lib.rs` via `tauri::generate_handler![]` with `commands::` prefix
- Tauri `invoke()` uses camelCase parameter names (e.g., `sessionId`, `rawContent`), which map to snake_case Rust params automatically
- `refreshItems()` pattern: call `invoke<T[]>('list_items')` after any mutation to keep UI in sync
- Theme store: import `theme` store and `initTheme()` from `$lib/stores/theme`
- Design tokens: use CSS custom properties like `var(--bg-primary)` everywhere
- PixelIcon component: `<PixelIcon name="folder" size={24} />` — import `IconName` type for type-safe icon names
- Sidebar component: `<Sidebar activeView={activeView} onnavigate={(view) => ...} />` — exports `ActiveView` type
- Svelte 5 event pattern: use `onnavigate` callback prop instead of `createEventDispatcher`
- All styles use CSS custom properties from tokens.css — no hardcoded colors
- Border-radius 0px throughout for pixel art sharpness; borders use `2px solid` for pixel feel
- FileCard exports `VaultItem` interface — import from `$lib/components/FileCard.svelte`
- FileBrowser component: callback props for events, renders empty state or file grid
- SearchBar: `<SearchBar bind:value={searchQuery} />` — uses `$bindable()` for two-way binding
- Center `<main>` has `position: relative` — required for absolute positioning overlays
- Center-area overlays: `position: absolute; inset: 0` constrained to center panel (not viewport)
- FileViewer exports `ItemDetail` interface — import from `$lib/components/FileViewer.svelte`
- EntityPanel exports `groupEntitiesByType()` — reusable entity grouping logic
- Svelte 5 `$derived.by(() => ...)` for complex derived values
- `<script lang="ts" module>` for exported types/functions importable from `.svelte` files
- Escape key dismissal: use `<svelte:window onkeydown={handler} />` for overlay components
- ConfirmDialog: callback props `onconfirm` and `oncancel`, constrained to center area
- `color-mix(in srgb, var(--bg-surface) 80%, transparent)` for semi-transparent backgrounds
- Crypto: `crypto::encrypt(key, plaintext)` and `crypto::decrypt(key, ciphertext)` — key from `CryptoState` managed state
- Encryption fallback: if decryption fails, return raw string (handles pre-encryption data)
- SessionList component: exports `SessionSummary` interface, uses `onselectsession` callback prop
- ConfirmDialog is rendered inside child components (not just +page.svelte) since overlay uses `position: absolute; inset: 0`
- SessionDetail component: `<SessionDetail sessionId={id} onback={() => ...} />` — loads session data on mount via `invoke('get_session')`
- SessionEntryCard component: `<SessionEntryCard entry={e} itemName={name} onselect={(id) => ...} />` — exports `SessionEntry` interface
- Output entries use `--color-grim-blue` accent, input entries use `--accent-orange` — consistent complementary color scheme
- Inline editing pattern: click text → show input, blur/enter saves, escape cancels — reuse for editable labels
- Frosted glass fixed-bottom: `position: absolute; bottom: 0; background: color-mix(in srgb, var(--bg-surface) 80%, transparent); backdrop-filter: blur(8px);` — parent needs `position: relative`, sibling scroll area needs bottom padding
- VaultPicker component: `<VaultPicker sessionItemIds={set} onselect={(id) => ...} onclose={() => ...} />` — self-contained, calls `invoke('list_items')` internally
- Session item linking flow: `add_item_to_session` → `reconcile_item_tokens` → `get_item` → `add_session_entry` (input type)
- Tauri drag-drop context routing: check `activeView` and `selectedSessionId` in the global drop handler to route to session-aware or browse-aware ingestion
- `export function` in Svelte 5 instance script exposes methods on component instance via `bind:this`
- SessionEntryPopup component: overlay with masked/decoded toggle and token highlighting
- Session entities for token mapping: `invoke('get_session_entities', { sessionId })` returns `SessionEntityOutput[]`
- MCP server: TypeScript package in `mirage-vault/mcp-server/` with dual transport (stdio + SSE)
- Desktop Extension packaging: `manifest.json` v0.3 format, `pnpm run package` builds and packs .mcpb, `.mcpbignore` excludes devDependencies
- `export_mcpb(dest_path)` Rust command: `find_mcpb_file()` follows same pattern as `find_mcp_script()` (env var → dev → production paths)
- `@tauri-apps/plugin-dialog` `save()` returns a file path — use with Rust command for file copy workflow
- Settings sections: use `.settings-section` class with `.section-label` h3 and `.subsection-label` h4 for nested groups
- SQLite INTEGER for booleans: read as i64, convert with `!= 0` for Rust bool fields
- MCP SDK imports: `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js`, `StdioServerTransport` from `@modelcontextprotocol/sdk/server/stdio.js`
- MCP SDK uses `server.registerTool()` with zod schemas for input validation
- MCP server tsconfig: target ES2022, module Node16, moduleResolution Node16, type: "module" in package.json; test files excluded from build
- Crypto compatibility: `deriveKey(passphrase)` → SHA-256, `decrypt(key, ciphertext)` → AES-256-GCM matching Rust format base64(nonce[12]||ciphertext||tag[16])
- `decryptSafe(key, ciphertext)` returns raw string on failure — use for DB fields that may be plaintext or encrypted
- MCP server `createServer(db?)` factory in `src/core/server.ts` — transport-agnostic, wired in index files; tools registered only when VaultDB provided
- MCP tool registration: `server.tool(name, description, { param: z.type() }, handler)` — handler returns `{ content: [{ type: 'text', text }] }`
- MCP tools file: `src/core/tools.ts` exports `registerTools(server, db)` — all tool registrations go here
- VaultDB initialized in index entry points from env vars MIRAGE_VAULT_PASSPHRASE and MIRAGE_VAULT_DB_PATH
- MCP tool access control: `get_entity_legend` uses `getSessionMcpStatus()` check; `get_masked_item` relies on VaultDB's SQL-level access check (EXISTS/NOT EXISTS pattern)
- MCP resources: `registerResources(server, db)` in `src/core/resources.ts` — wired in `server.ts` alongside `registerTools()`
- MCP ResourceTemplate: `new ResourceTemplate(uriTemplate, { list: callback })` — list callback returns available resources for template URIs
- MCP resource handler receives `(uri: URL, variables: Variables)` — variables extracted from URI template; use `Number(variables.session_id)` to parse
- better-sqlite3 needs native build — use `pnpm.onlyBuiltDependencies` in package.json
- VaultDB class in `src/core/db.ts`: readonly better-sqlite3 connection, takes `(key, dbPath?)` — default path based on `process.platform`
- `getSessionMcpStatus(sessionId)` lightweight check: returns `{ id, mcp_shared }` without loading full session data — use for access control in tools
- Session entry `raw_content` is encrypted — use `decryptSafe(key, row.raw_content)` when reading
- Items `masked_content` is NOT encrypted — read directly from DB
- Entity `original_value` is encrypted but NEVER exposed via MCP — only serve `entity_type` and `token`
- `getMaskedItem` access check: item must be in an mcp_shared session OR not in any session (EXISTS/NOT EXISTS pattern)
